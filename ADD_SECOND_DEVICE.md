# Добавление второго устройства Яндекс.Go

## Шаг 1: Получить credentials со второго устройства

На втором телефоне/эмуляторе:

1. Установить Яндекс.Go
2. Зарегистрировать новый аккаунт
3. Перехватить запросы (через Charles Proxy / HTTP Toolkit)
4. Найти headers:
   - `Authorization: Bearer XXX` → bearer_token
   - `x-appmetrica-uuid: XXX` → uuid
   - `x-appmetrica-deviceid: XXX` → device_id
   - `x-mob-id: XXX` → mob_id

## Шаг 2: Добавить в .env на сервере

Открыть `/opt/taxibot/.env` и добавить второе устройство через `|`:

```bash
# Было (1 устройство):
YANDEX_BEARER_TOKEN=token1
YANDEX_UUID=uuid1
YANDEX_DEVICE_ID=device1
YANDEX_MOB_ID=mob1

# Стало (2 устройства):
YANDEX_BEARER_TOKENS=token1|token2
YANDEX_UUIDS=uuid1|uuid2
YANDEX_DEVICE_IDS=device1|device2
YANDEX_MOB_IDS=mob1|mob2
```

**Важно:** Используй `YANDEX_BEARER_TOKENS` (с S) для множественных устройств!

## Шаг 3: Добавить новые зоны

После добавления второго устройства можно безопасно добавить **+71 зону**.

### Откуда взять зоны?

**Вариант 1: Расширить покрытие Москвы**
- Добавить районы на окраинах
- Заполнить пробелы между существующими зонами
- Добавить спальные районы

**Вариант 2: Добавить Подмосковье**
- Химки, Мытищи, Балашиха
- Люберцы, Реутов, Королёв
- Одинцово, Красногорск

**Вариант 3: Добавить другие города**
- Санкт-Петербург (топ районы)
- Казань, Екатеринбург, Новосибирск
- Нижний Новгород, Самара

### Как добавить зоны в код:

Редактировать `bot/services/zones.py`:

```python
def get_zones() -> list[Zone]:
    zones = [
        # Существующие 89 зон...

        # Новые зоны (добавить +71):
        Zone("novye_cheremushki", "Новые Черёмушки", 55.6708, 37.5561),
        Zone("yasenevo_south", "Ясенево Юг", 55.6000, 37.5333),
        # ... ещё 69 зон
    ]
    return zones
```

## Шаг 4: Тестирование

После добавления второго устройства:

```bash
# Перезапустить бота
cd /opt/taxibot && ./update_bot.sh

# Проверить логи
tail -f bot.log | grep "Loaded.*device credentials"
```

Должно быть:
```
INFO: Loaded 2 device credentials for load distribution
```

## Шаг 5: Мониторинг

Следить за 429 ошибками первые 24 часа:

```bash
# Проверить ошибки
grep -i '429' bot.log | grep "$(date +%Y-%m-%d)" | wc -l

# Должно быть: 0 или минимум
```

Если появятся 429 ошибки:
1. Уменьшить количество зон
2. Увеличить интервал парсинга до 10 минут

## Преимущества 2 устройств

✅ **Нагрузка распределяется**
- Каждое устройство: ~240 запросов вместо 267
- Снижение риска rate limit

✅ **Можно добавить +71 зону**
- Лучшее покрытие Москвы
- Больше данных для пользователей

✅ **Отказоустойчивость**
- Если одно устройство заблокируют → второе работает
- Система продолжит работать

## Риски

⚠️ **Оба устройства могут заблокировать**
- Если нагрузка слишком высокая
- Если Яндекс обнаружит паттерн

⚠️ **Нужно поддерживать 2 аккаунта**
- Токены могут истечь
- Нужно обновлять credentials
