# Changelog - Taxi Bot

## [1.1.0] - 2026-03-19

### ✨ Добавлено

#### Спортивные мероприятия
- Парсинг спортивных событий из KudaGo API (категория "sport")
- Парсинг выставок и фестивалей (категории "exhibition", "festival")
- Расширенное определение типа события: 40+ ключевых слов для спорта
- Поддержка названий команд: Спартак, ЦСКА, Динамо, Локомотив, Зенит
- Поддержка турниров: РФПЛ, КХЛ, ВТБ, Евролига, Лига Чемпионов

#### Дополнительные источники событий
- Базовая структура парсера Sports.ru (готова к реализации)
- Базовая структура парсера Kassir.ru (готова к реализации)
- Архитектура для парсинга из нескольких источников одновременно
- Система дедупликации событий из разных источников

#### Стадионы
- Добавлено 3 новых стадиона в venues.json:
  - Арена Мытищи (хоккей)
  - Крылатское (велотрек)
  - Дворец Спорта Янтарь
- Расширены алиасы существующих стадионов

### 🔧 Изменено

#### Парсер событий
- Обновлена длительность спортивных событий: 2.5 часа (было 2 часа)
- Улучшена функция `_guess_event_type()` с расширенным списком ключевых слов
- Функция `parse_kudago_events()` теперь парсит 4 категории вместо 1
- Функция `fetch_and_store_events()` поддерживает множественные источники

#### Логирование
- Добавлено логирование по категориям событий
- Улучшены сообщения об ошибках парсинга
- Добавлена статистика по источникам

### 📚 Документация

Созданы новые файлы документации:
- `SPORT_EVENTS_UPDATE.md` - описание обновления
- `ADDITIONAL_SOURCES.md` - список дополнительных источников
- `EVENT_SOURCES_CONFIG.md` - конфигурация источников
- `TESTING_CHECKLIST.md` - чеклист тестирования
- `DEPLOY_INSTRUCTIONS.md` - инструкция по деплою
- `CHANGELOG.md` - этот файл

### 🐛 Исправлено
- Улучшена обработка ошибок при парсинге событий
- Добавлена проверка на дубликаты событий

### 🔒 Безопасность
- Добавлены реалистичные User-Agent для парсинга
- Добавлены задержки между запросами (rate limiting)
- Graceful degradation при падении источников

## Формат версий

Проект использует [Semantic Versioning](https://semver.org/):
- MAJOR версия - несовместимые изменения API
- MINOR версия - новая функциональность (обратно совместимая)
- PATCH версия - исправления ошибок

## Типы изменений

- ✨ **Добавлено** - новая функциональность
- 🔧 **Изменено** - изменения в существующей функциональности
- 🗑️ **Устарело** - функциональность, которая скоро будет удалена
- ❌ **Удалено** - удаленная функциональность
- 🐛 **Исправлено** - исправления ошибок
- 🔒 **Безопасность** - исправления уязвимостей
