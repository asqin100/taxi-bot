## ✅ Парсинг мероприятий (KudaGo API)

**Интеграция завершена!** Бот теперь автоматически загружает мероприятия из KudaGo API.

- **API:** https://kudago.com/public-api/v1.4/
- **Endpoint:** `/events/?location=msk&categories=concert,sport`
- **Параметры:**
  - Локация: Москва
  - Категории: концерты, спорт
  - Период: следующие 30 дней
  - Лимит: 100 событий за запрос
- **Автообновление:** каждые 6 часов
- **Маппинг:** события автоматически привязываются к 31 площадке

**Как работает:**
1. Scheduler запускает `fetch_and_store_events()` каждые 6 часов
2. Функция `parse_kudago_events()` получает события из API
3. Для каждого события определяется площадка и зона
4. События сохраняются в БД (дубликаты пропускаются)
5. Система уведомлений отслеживает время окончания

## 🐛 Известные проблемы

1. **Rate limiting (429)** от Яндекс API
   - Решение: 5 секунд между запросами + retry логика
   - Работает стабильно

2. **Парсер Яндекс.Афиши не работает** ✅ РЕШЕНО
   - Причина: SPA с динамической загрузкой
   - Решение: интегрирован KudaGo API

3. **Ngrok URL может измениться**
   - При перезапуске ngrok генерирует новый URL
   - Нужно обновить WEBAPP_URL в .env
