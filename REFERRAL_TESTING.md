# Тестирование реферальной системы

## Быстрый старт

### 1. Запуск бота
```bash
cd C:\taxi-bot
python start_bot.py
```

### 2. Базовое тестирование

#### Шаг 1: Получить реферальную ссылку
1. Отправьте боту `/referral`
2. Скопируйте вашу реферальную ссылку
3. Проверьте, что отображается баланс (0₽ для нового пользователя)

#### Шаг 2: Зарегистрировать реферала
1. Откройте реферальную ссылку в другом аккаунте Telegram
2. Нажмите "Start"
3. Проверьте, что новый пользователь зарегистрирован

#### Шаг 3: Проверить начисление бонусов (требует тестовую оплату)
1. От имени реферала купите подписку (Pro или Premium)
2. Вернитесь к первому аккаунту
3. Отправьте `/referral` и проверьте:
   - Баланс увеличился на 30% от стоимости подписки
   - Если это первая оплата реферала, добавлено +100₽
   - Счетчик рефералов увеличился

#### Шаг 4: Оплата с реферального баланса
1. Накопите достаточно средств (299₽ для Pro или 499₽ для Premium)
2. Нажмите "💰 Оплатить подпиской"
3. Выберите тариф
4. Проверьте, что:
   - Подписка активирована
   - Баланс уменьшился на стоимость подписки
   - Подписка отображается в "⭐ Подписка"

```sql
-- Проверить пользователей с реферальными кодами
SELECT telegram_id, username, referral_code, referrer_id, referral_balance
FROM users
WHERE referral_code IS NOT NULL;

-- Проверить историю начислений
SELECT * FROM referral_earnings ORDER BY created_at DESC LIMIT 10;

-- Проверить реферальную структуру
SELECT
    u1.telegram_id as referrer,
    u1.referral_balance as balance,
    COUNT(u2.id) as referrals_count
FROM users u1
LEFT JOIN users u2 ON u2.referrer_id = u1.id
WHERE u1.referral_code IS NOT NULL
GROUP BY u1.id;
```

## Тестовые сценарии

### Сценарий 1: Базовая реферальная цепочка
1. Пользователь A получает реферальную ссылку
2. Пользователь B регистрируется по ссылке A
3. Пользователь B покупает Pro (299₽)
4. Проверить: A получил 89.7₽ (30%) + 100₽ (бонус) = 189.7₽

### Сценарий 2: Двухуровневая структура
1. Пользователь A приглашает B
2. Пользователь B приглашает C
3. Пользователь C покупает Premium (499₽)
4. Проверить:
   - B получил 149.7₽ (30%) + 100₽ (бонус) = 249.7₽
   - A получил 49.9₽ (10%)

### Сценарий 3: Milestone бонусы
1. Пользователь A приглашает 5 рефералов
2. Проверить: A получил +200₽ бонус
3. Пользователь A приглашает еще 5 (всего 10)
4. Проверить: A получил +500₽ бонус

### Сценарий 4: Оплата с баланса
1. Пользователь A накопил 299₽
2. A оплачивает Pro с реферального баланса
3. Проверить:
   - Баланс A уменьшился на 299₽
   - Подписка активирована на 30 дней
   - В истории есть запись WITHDRAWAL на -299₽

## Проверка безопасности

### Тест 1: Самореферальство
1. Получите свою реферальную ссылку
2. Попробуйте использовать её в том же аккаунте
3. Ожидаемый результат: Ошибка или игнорирование

### Тест 2: Повторная регистрация
1. Зарегистрируйте пользователя B по ссылке A
2. Попробуйте снова зарегистрировать B по другой ссылке
3. Ожидаемый результат: Игнорирование (referrer_id уже установлен)

### Тест 3: Недостаточный баланс
1. Попробуйте купить подписку с балансом < стоимости
2. Ожидаемый результат: Ошибка "Недостаточно средств"

## Логирование

Проверьте логи для отладки:
```bash
tail -f C:\taxi-bot\bot.log | grep -i referral
```

Ключевые события в логах:
- `Generated referral code` - создан новый код
- `User X registered as referral of Y` - регистрация реферала
- `Level 1: User X earned Y₽` - начисление уровня 1
- `Level 2: User X earned Y₽` - начисление уровня 2
- `First subscription bonus` - бонус за первую оплату
- `Milestone bonus` - бонус за количество
- `User X withdrew Y₽` - списание с баланса

## Известные ограничения

1. **Реферальные коды генерируются лениво** - только при первом обращении к `/referral`
2. **Бонусы начисляются только при оплате через YooKassa** - оплата с баланса не генерирует новые бонусы
3. **Milestone бонусы начисляются один раз** - повторное начисление невозможно
4. **Нет автоматического вывода средств** - баланс можно использовать только для оплаты подписки

## Troubleshooting

### Проблема: Реферальный код не создается
**Решение:** Проверьте, что миграция выполнена (`python migrate_referral.py`)

### Проблема: Бонусы не начисляются
**Решение:**
1. Проверьте логи на ошибки
2. Убедитесь, что оплата прошла через YooKassa webhook
3. Проверьте, что у реферала установлен referrer_id

### Проблема: Не работает оплата с баланса
**Решение:**
1. Проверьте баланс пользователя в БД
2. Убедитесь, что баланс >= стоимости подписки
3. Проверьте логи на ошибки при списании
