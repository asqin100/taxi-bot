# 📋 РУКОВОДСТВО: Добавление спортивных матчей

## Проблема
Большинство спортивных сайтов (РПЛ, КХЛ, Kassir.ru) защищены от автоматического парсинга или загружают данные через JavaScript, что делает автоматический парсинг невозможным.

## Решение
Используем **ручное добавление** важных матчей через JSON файл.

---

## Способ 1: Через JSON файл (рекомендуется для массового добавления)

### Шаг 1: Отредактируй файл `data/manual_sports_matches.json`

Добавь новые матчи в массив `upcoming_matches`:

```json
{
  "name": "Спартак - Зенит",
  "sport": "football",
  "venue": "Спартак Стадион",
  "venue_lat": 55.8181,
  "venue_lon": 37.4403,
  "zone_id": "northwest",
  "date": "2026-03-30",
  "time": "19:00",
  "league": "РПЛ"
}
```

### Шаг 2: Загрузи матчи в базу данных

На сервере выполни:
```bash
cd /opt/taxibot
source venv/bin/activate
python import_manual_matches.py
```

---

## Способ 2: Через админ панель (для единичных матчей)

1. Открой админ панель
2. Перейди в раздел **Мероприятия**
3. Нажми **➕ Добавить мероприятие**
4. Заполни форму:
   - **Название**: "Спартак - ЦСКА"
   - **Зона**: northwest (для Спартак Стадион)
   - **Тип**: sport
   - **Время окончания**: 30.03 21:30 (матч в 19:00 + 2.5 часа)

## Рекомендации

1. **Обновляй расписание каждый понедельник** - добавляй матчи на следующие 2 недели
2. **Фокусируйся на домашних матчах** московских команд - именно они создают спрос на такси
3. **Добавляй важные матчи**: дерби, плей-офф, финалы
4. **Время окончания** = время начала + 2.5 часа (для футбола/хоккея/баскетбола)

---

## Автоматизация (опционально)

Можно настроить cron job для еженедельного напоминания:

```bash
# Каждый понедельник в 10:00
0 10 * * 1 echo "Обнови расписание матчей в manual_sports_matches.json" | mail -s "Taxi Bot: Update Sports Schedule" admin@example.com
```

---

## Проверка

После добавления матчей проверь в админ панели:
1. Открой **Мероприятия**
2. Нажми фильтр **⚽ Спорт**
3. Должны отобразиться все добавленные матчи
