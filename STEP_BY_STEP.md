# Пошаговая инструкция по исправлению webhook

## Проблема
Платеж проходит, но Robokassa не вызывает Result URL, поэтому подписка не активируется.

## Решение

### Шаг 1: Сохраните Result URL в Robokassa

1. Откройте https://auth.robokassa.ru/
2. Перейдите в настройки магазина **kefpulse**
3. Откройте **"Технические настройки"**
4. Найдите раздел для **БОЕВОГО режима** (не тестового!)
5. В поле **Result URL** введите:
   ```
   http://5.42.110.16:8080/webhook/robokassa/result
   ```
6. Убедитесь, что метод: **GET** (не POST)
7. **ОБЯЗАТЕЛЬНО нажмите "Сохранить"!**
8. Сделайте скриншот сохраненных настроек

### Шаг 2: Запустите мониторинг на сервере

Выполните на сервере:
```bash
cd /opt/taxibot && git pull origin main && bash monitor_webhook.sh
```

Эта команда покажет логи в реальном времени. Оставьте терминал открытым.

### Шаг 3: Сделайте тестовый платеж

1. Откройте бота @KefPulse_bot
2. Перейдите: /menu → 💎 Подписка → ⬆️ Улучшить тариф
3. Нажмите: 🧪 ТЕСТ — 5₽ (1 день)
4. Оплатите 5 рублей

### Шаг 4: Проверьте логи

Сразу после оплаты в терминале с мониторингом должны появиться строки:
```
Received Robokassa Result callback: InvId=...
```

Если эти строки появились:
- ✅ Webhook работает!
- ✅ Подписка должна активироваться
- ✅ Придет уведомление в бот

Если строк НЕТ:
- ❌ Result URL все еще не сохранен в Robokassa
- Повторите Шаг 1, убедитесь что нажали "Сохранить"

### Шаг 5: Остановите мониторинг

Нажмите `Ctrl+C` в терминале для остановки мониторинга.

---

## Частые ошибки

1. **Result URL сохранен только для тестового режима**
   - В Robokassa есть ДВА набора настроек: для теста и для боевого
   - Нужно сохранить URL в ОБОИХ разделах

2. **Забыли нажать "Сохранить"**
   - После ввода URL обязательно нажмите кнопку "Сохранить"

3. **Неправильный метод**
   - Должен быть GET, а не POST

---

## Если ничего не помогает

Пришлите скриншот страницы технических настроек Robokassa (раздел боевого режима).
