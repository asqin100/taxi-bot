# Чеклист тестирования спортивных событий

## ✅ Перед деплоем

- [ ] Проверить синтаксис Python: `python3 -m py_compile bot/services/event_parser.py`
- [ ] Убедиться, что все импорты на месте
- [ ] Проверить, что venues.json содержит стадионы
- [ ] Сделать бэкап базы данных

## ✅ После деплоя

### Парсинг событий

- [ ] Проверить логи: `tail -f bot.log | grep "sport"`
- [ ] Убедиться, что парсятся оба типа: concert и sport
- [ ] Проверить, что события сохраняются в БД
- [ ] Проверить, что не создаются дубликаты

### Команды бота

- [ ] `/events` - показывает спортивные события с эмодзи ⚽
- [ ] `/addevent` - можно добавить событие типа "sport"
- [ ] Проверить отображение зон для стадионов

### Уведомления

- [ ] Настройки → Уведомления → "⚽ Спорт" доступен
- [ ] Можно включить/выключить уведомления о спорте
- [ ] Уведомления приходят за 20 минут до окончания (Pro+)
- [ ] Уведомления приходят после окончания (все)
- [ ] Эмодзи ⚽ отображается в уведомлениях
- [ ] Кнопка "🚕 Поехать" работает (навигация к стадиону)

### Определение типа события

Проверить, что правильно определяются:
- [ ] "Спартак - ЦСКА" → sport
- [ ] "Матч КХЛ: Динамо - ЦСКА" → sport
- [ ] "Концерт в Лужниках" → concert (не sport!)
- [ ] "Футбольный турнир" → sport
- [ ] "Баскетбол ВТБ" → sport

### Стадионы и зоны

- [ ] Лужники → zone: south
- [ ] Спартак Стадион → zone: northwest
- [ ] ЦСКА Арена → zone: northwest
- [ ] Локомотив Стадион → zone: northeast
- [ ] ВТБ Арена → zone: north

### Граничные случаи

- [ ] События без указания стадиона (должны пропускаться)
- [ ] События в прошлом (должны игнорироваться)
- [ ] События с неизвестным стадионом (должны пропускаться)
- [ ] Дубликаты событий (не должны создаваться)

## ✅ Производительность

- [ ] Парсинг не занимает больше 30 секунд
- [ ] Нет утечек памяти
- [ ] Логи не переполняются
- [ ] БД не растет слишком быстро

## ✅ Пользовательский опыт

- [ ] Уведомления приходят вовремя
- [ ] Текст уведомлений понятен
- [ ] Кнопки работают
- [ ] Нет спама уведомлениями
- [ ] Тихие часы работают

## 🐛 Известные проблемы

_Заполнить после тестирования_

## 📊 Метрики для отслеживания

- Количество спортивных событий в БД
- Количество отправленных уведомлений
- Процент успешных парсингов
- Количество пользователей с включенными уведомлениями о спорте
