ПРОБЛЕМА:
Мероприятия показывались как завершенные на 3 часа раньше.
Пример: событие в 19:00 МСК отображалось как завершенное в 16:00.

ПРИЧИНА:
KudaGo API возвращает время в UTC (всемирное координированное время).
Код обрабатывал эти timestamps как локальное время без конвертации.
Разница: UTC + 3 часа = МСК

РЕШЕНИЕ:
✅ Добавлен импорт ZoneInfo для работы с часовыми поясами
✅ Создана функция get_moscow_now() для получения текущего времени МСК
✅ UTC timestamps из KudaGo конвертируются в МСК
✅ Все сравнения времени используют московское время

ИЗМЕНЕНИЯ В КОДЕ:

Конвертация времени:
1. KudaGo API возвращает: 1710691200 (Unix timestamp в UTC)
2. Конвертируем в UTC datetime: 2026-03-17 16:00:00 UTC
3. Конвертируем в MSK datetime: 2026-03-17 19:00:00 MSK
4. Сохраняем как naive: 2026-03-17 19:00:00 (БД предполагает МСК)

Все сравнения теперь используют московское время:
- get_moscow_now() возвращает текущее время в МСК
- Event.end_time хранится в МСК (naive datetime)
- Сравнение: end_time > get_moscow_now() работает корректно
