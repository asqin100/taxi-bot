## Как обновить бот на сервере

### Вариант 1: SSH доступ к серверу

```bash
# Подключиться к серверу
ssh user@your-server

# Перейти в папку с ботом
cd /path/to/taxi-bot

# Получить изменения
git pull origin main

# Перезапустить бот (зависит от способа запуска)
# Если systemd:
sudo systemctl restart taxi-bot

# Если screen/tmux:
# Найти процесс и убить его, затем запустить заново
pkill -f "python -m bot.main"
python -m bot.main &

# Если supervisor:
supervisorctl restart taxi-bot
```

### Вариант 2: Через панель управления хостингом

1. Зайти в панель управления (например, cPanel, Plesk)
2. Открыть Git Manager или File Manager
3. Выполнить `git pull` или загрузить файлы вручную
4. Перезапустить приложение через Application Manager

### Вариант 3: Автоматическое обновление (если настроено)

Если настроен webhook или CI/CD, бот обновится автоматически после push в GitHub.

## Проверка работы

После обновления проверить:
1. Открыть бот в Telegram
2. Перейти в "Подписка" → "Улучшить тариф"
3. Убедиться, что появилась кнопка "🧪 Тест платеж 5₽ (1 день Pro)"
4. Протестировать оплату 5₽ через Robokassa
5. Проверить, что приходит чек (задача #9)

## Логи

Проверить логи на наличие ошибок:
```bash
# Если systemd:
journalctl -u taxi-bot -f

# Если файл логов:
tail -f /path/to/bot.log
```
