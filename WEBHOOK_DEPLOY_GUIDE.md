2. **Создан скрипт для тестирования** - `test_webhook.py` для проверки webhook локально
3. **Упрощен процесс развертывания** - `deploy_webhook_fix.sh`

## Тестирование webhook

### 1. Локальное тестирование (на сервере)

```bash
cd /opt/taxibot
source venv/bin/activate
python test_webhook.py YOUR_TELEGRAM_ID
```

Замените `YOUR_TELEGRAM_ID` на ваш реальный Telegram ID.

Скрипт проверит:
- ✅ Настройки Robokassa
- ✅ Генерацию и проверку подписи
- ✅ Обработку платежа
- ✅ Активацию подписки

### 2. Проверка через реальный платеж

1. Откройте бота в Telegram
2. Выберите подписку (например, Pro)
3. Нажмите "Оплатить картой"
4. Используйте тестовую карту: **5555 5555 5555 5599**
5. После оплаты должно произойти:
   - ✅ Перенаправление на страницу успеха
   - ✅ Получение сообщения в боте: "✅ Подписка активирована!"
   - ✅ Подписка активна в /menu

## Диагностика проблем

### Если бот не отправляет уведомление:

```bash
# Проверить логи бота
journalctl -u taxibot --since "5 minutes ago" -f

# Проверить, что webhook получен
grep "Robokassa Result callback" /var/log/syslog

# Проверить, что бот запущен
systemctl status taxibot

# Проверить, что порт 8080 слушает
netstat -tlnp | grep 8080
```

### Если webhook не приходит от Robokassa:

1. **Проверьте Result URL в личном кабинете Robokassa:**
   - Должен быть: `http://5.42.110.16:8080/webhook/robokassa/result`
   - Метод: GET
   - Не забудьте нажать "Сохранить"!

2. **Проверьте Success URL:**
   - `http://5.42.110.16:8080/webhook/robokassa/success`

3. **Проверьте Fail URL:**
   - `http://5.42.110.16:8080/webhook/robokassa/fail`

### Проверка доступности webhook извне:

```bash
# С другого компьютера или через онлайн-сервис
curl "http://5.42.110.16:8080/webhook/robokassa/result?test=1"
```

Должен вернуть ответ (даже если это ошибка обработки).

## Что должно произойти после оплаты

1. **Robokassa** обрабатывает платеж
2. **Robokassa** отправляет GET-запрос на Result URL
3. **Бот** получает webhook, проверяет подпись
4. **Бот** активирует подписку в базе данных
5. **Бот** начисляет реферальные бонусы (если есть)
6. **Бот** отправляет сообщение пользователю в Telegram
7. **Бот** возвращает "OK{InvId}" Robokassa
8. **Пользователь** видит страницу успеха в браузере

## Частые проблемы

### 1. "ERROR" вместо "OK{InvId}"
- Неверная подпись (проверьте Password2)
- Неверная сумма платежа
- Ошибка в базе данных

### 2. Webhook не приходит
- Result URL не сохранен в Robokassa
- Порт 8080 закрыт файрволом
- Бот не запущен

### 3. Подписка не активируется
- Проверьте логи: `journalctl -u taxibot -n 100`
- Проверьте базу данных: есть ли пользователь с таким telegram_id

## Контакты для поддержки
