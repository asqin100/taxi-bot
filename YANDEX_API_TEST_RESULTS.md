# Результаты тестирования Yandex API

## Проблема с ключом

API ключ `e1a5f0ce-7744-4c0f-9c3d-34c9288b9a42` не работает:

```
Geocoder API: 403 "Invalid api key"
Router API v2: 401 "Apikey rejected"
```

## Причина

**"Маршрутизация: Планирование и Мониторинг"** - это НЕ API для получения данных о пробках.

Это сервис для **логистики и доставок**:
- Планирование маршрутов курьеров
- Оптимизация доставок
- Мониторинг выполнения заказов

Он предназначен для компаний-доставщиков (Яндекс.Еда, Delivery Club и т.д.), а не для получения данных о трафике.

## Вывод

У Яндекса **НЕТ публичного REST API** для получения уровня пробок, который можно использовать из Python бэкенда.

Доступные API Яндекса:
- ❌ JavaScript API - только браузер
- ❌ Tiles API - картинки, не данные
- ❌ HTTP Геокодер - только адреса
- ❌ Маршрутизация - логистика для доставок
