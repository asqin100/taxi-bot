# Исследование: Как получить данные о пробках из Яндекса

## Проблема
У Яндекса нет публичного REST API для получения уровня пробок (traffic level 1-10), как у TomTom.

## Доступные API в кабинете разработчика

### ❌ Не подходят:
- **JavaScript API** - только для браузера
- **Tiles API** - картинки слоя пробок (нельзя извлечь число)
- **HTTP Геокодер** - только адреса → координаты
- **Маршрутизация: Планирование и Мониторинг** - логистика для доставок

### ⚠️ Возможно подходит:
- **Router API** - строит маршруты с учетом пробок
  - Возвращает duration (время в пути)
  - Время учитывает текущие пробки
  - Можно попробовать вывести уровень пробок из времени

## Идея решения через Router API

### Концепция:
1. Запрашиваем маршрут между двумя точками
2. Получаем duration с учетом пробок
3. Сравниваем с базовым временем (без пробок)
4. Вычисляем коэффициент: `traffic_ratio = actual_duration / base_duration`
5. Конвертируем в уровень 1-10

### Пример:
```
Маршрут: Центр → Север (10 км)
Базовое время: 15 минут (без пробок)
Текущее время: 30 минут (с пробками)
Коэффициент: 30/15 = 2.0
Уровень пробок: 8/10 (серьезные пробки)
```

### Проблемы:
- Нужно делать запрос для каждой точки (расход API лимита)
- Нужно знать базовое время без пробок
- Менее точно, чем прямой Traffic API
